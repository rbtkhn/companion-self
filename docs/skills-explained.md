# Skills explained — two layers (companion-self)

**Purpose:** Disambiguate **operator / Cursor skills** (portable execution assets) from **SKILLS** (Record capability index). The template uses the same **governance** idea as instances; file layouts may vary.

> **SKILLS** names a **Record surface** (capability-facing). **Portable skills** name reusable **operator** assets — often in `.cursor/skills/` or a repo-level `skills-portable/` directory **when an instance adds them** (see Grace-Mar).

---

## Portable skills (operator / execution)

- **Template:** `.cursor/` may ship **skills** for **template contributors** (see [README.md](../README.md) § Contents).
- **Role:** Procedural helpers — they **stage** work; they do **not** merge the Record.

---

## SKILLS (Record — capability)

- **On disk (typical instance):** `self-skills.md` (capability index), aligned with [layer-architecture.md](layer-architecture.md) and instance doctrine.
- **Governance:** Updates that change durable capability claims follow the **gate** / change-review path for that instance — not silent edits.
