# Work modules — authorized sources lists

**Design principle:** Each **work** territory (`work-*`) may carry a single hand-maintained file naming **authorized sources** — channels, podcasts, newsletters, or principal social accounts that **orient** operator work in that lane (framing, cadence, provenance).

| Convention | Meaning |
|--------------|---------|
| **Filename** | `<territory-id>-sources.md` (e.g. `work-dev-sources.md`) |
| **Location** | `docs/skill-work/<territory>/` alongside that territory’s README |
| **Content** | Short table: source name, URL, optional notes. Add rows manually. |
| **Not** | Record truth, Voice knowledge, or automatic ingest config. |

**Fence:** These lists document **where ideas and vocabulary often come from**, not what is true.
