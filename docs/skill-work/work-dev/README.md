# work-dev (template module)

**Status:** Empty starter module in `companion-self`.

**Purpose:** WORK territory for agent operations, reliability practices, sync discipline, and process hardening.

---

## How this gets populated

Initial population should come from the instance's **seed survey** (tool literacy, operating preferences, risk style) and early onboarding observations.

Typical first docs:

- `operating-principles.md`
- `reliability-checklist.md`
- `sync-notes.md`

---

## Daily usage

Use in `good morning` to:

- run daily relevance/sync checks
- review reliability guardrails
- select one process-improvement action for the day

Keep all edits in WORK surfaces; stage identity implications through gate flow.
