# Lawful Succession

Companion-Self treats a long-lived companion as something that must preserve intelligible continuity across change.

The system does not treat every meaningful revision as a routine profile update. When a proposed post-seed change materially affects identity, curiosity, pedagogy, expression, memory governance, safety rules, or other durable operating commitments, the change should be interpreted not only as a supported update, but as a possible **succession event**.

This document introduces the constitutional layer for that interpretation.

## Why this layer exists

Memory alone is not enough for long-lived companions.

A system can remember accurately and still lose continuity if:
- major changes are merged without preserving prior state
- repeated contradictions accumulate without interpretation
- durable commitments change faster than the review process can explain
- the future self becomes difficult to understand as a lawful successor to the prior one

The goal of lawful succession is to preserve a readable history of becoming.

## Core claim

The central problem of long-lived companion systems is not only memory retention.

It is the legitimacy of change over time.

Companion-Self therefore distinguishes between:
- ordinary staged updates
- materially important governed changes
- constitutional-level transitions affecting the self's continuity

## Succession event

A **succession event** is a materially important post-seed change that could alter how the companion:
- understands itself
- teaches
- speaks
- remembers
- governs future merges
- maintains boundary or safety doctrine

Not every gated item is a succession event.

A succession event occurs when a proposed change meaningfully affects durable governed state.

## What lawful succession requires

For a future self to count as a lawful successor, change should preserve:

1. **Prior state visibility**  
   The earlier governed state must remain inspectable.

2. **Process visibility**  
   The path from proposal to decision must remain reviewable.

3. **Evidence basis**  
   Material changes should be supported by evidence, not merely by drift or convenience.

4. **Approval integrity**  
   Governed state should not change through silent overwrite.

5. **Continuity intelligibility**  
   A reader should be able to explain how the later state emerged from the earlier one.

## Contradiction as pressure

Contradiction is not only an error condition.

In long-lived governed systems, contradiction can also be evidence of:
- unresolved duality
- reform pressure
- boundary ambiguity
- legitimacy decay
- branch pressure

Companion-Self should preserve contradictions long enough to interpret what they mean for continuity.

## Review tiers

The constitutional layer introduces three review tiers:

### 1. Gate
Default staging surface for ordinary proposals.

### 2. Change review
Required for materially important governed changes, as already defined in the change-review doctrine.

### 3. Constitutional review
Required when a proposal may alter the lawful continuity of the self in a structural way.

Examples include:
- durable identity shifts
- memory-governance changes
- safety or boundary doctrine changes
- major pedagogy doctrine changes
- major expression doctrine changes
- changes that collide with prior constitutional commitments

## What this document does not do

This document does not yet define:
- machine-readable schemas
- scoring thresholds
- validator rules
- instance-specific implementation details

Those belong in later layers after the constitutional vocabulary is stable.

## Relationship to existing doctrine

This document extends, but does not replace:
- Seed Phase doctrine
- governed post-seed change doctrine
- contradiction policy
- change-review lifecycle

The constitutional layer asks an additional question:

**Is the proposed future state a lawful successor to the prior one?**
