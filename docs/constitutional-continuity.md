# Constitutional Continuity

This document defines the tests used to judge whether a proposed future state remains a lawful successor to the prior one.

## Purpose

A companion can change substantially while remaining continuous.

The question is not whether change exists, but whether the change remains intelligible, reviewable, and legitimate.

## Continuity tests

### 1. Prior-state preservation

The previous governed state must remain inspectable.

A reader should be able to see what the system was before the proposed transition.

### 2. Proposal traceability

The source of the proposed change must be visible.

A reviewer should be able to trace the proposal to staged items, evidence, review artifacts, or constitutional review materials.

### 3. Evidence adequacy

The proposal should be supported in proportion to its depth.

The deeper the transition, the higher the burden to show:
- repeated evidence
- direct evidence
- durable behavioral support
- explicit user approval where required

### 4. Process integrity

The proposal must use the correct approval path.

Material changes should not bypass governed review.

### 5. Continuity intelligibility

A reviewer should be able to explain how the proposed state emerged from the prior state without resorting to hand-waving or silent overwrite.

### 6. Contradiction burden

The proposal should identify whether it:
- resolves contradiction
- preserves contradiction
- introduces contradiction
- escalates contradiction into structural pressure

### 7. Reversibility awareness

The proposal should acknowledge whether it can be safely reversed.

Low-reversibility changes carry a higher continuity burden.

### 8. Branch legitimacy

When continuity cannot be preserved in one line without distortion, branching may be more lawful than replacement.

## Interpretive rule

A lawful successor need not be identical to the prior state.

It must, however, remain:
- historically readable
- procedurally grounded
- evidentially supported
- structurally intelligible

## Failure modes

Continuity weakens when:
- prior state is overwritten
- process is hidden
- evidence is thin relative to impact
- contradictions accumulate without interpretation
- branch pressure is denied
- governance rules conflict without resolution

## Relationship to classification

These continuity tests are used together with regime classification.

For example:
- an administrative update may pass continuity tests with light review
- an amendment requires stronger continuity proof
- a fork may preserve continuity better than a forced merge
- coup-risk and collapse-risk often indicate continuity failure
