# Good Night Template

This document defines the recommended **Good Night** sequence for companion-self instances.

Goal: a short, low cognitive load closeout (2-4 minutes) that captures high-leverage signals for tomorrow without creating heavy end-of-day work.

---

## Design principles

- **Short by default**: should complete in a few minutes.
- **Low cognitive load**: mostly checkboxes and one-line responses.
- **High leverage**: one carry-forward decision beats long journaling.
- **Safe**: no direct identity merges; gate discipline remains intact.

---

## Core flow (recommended)

### 1) Close loop check (30-60s)

Mark today as:

- finished well
- partial progress
- blocked

Include one line: what was completed or what blocked progress.

### 2) One signal capture (45-90s)

Capture exactly one high-value signal:

- a learning signal (think/write/work)
- a risk/boundary signal
- a process signal (what worked or failed)

Keep it to one short bullet.

### 3) Tomorrow handoff seed (45-90s)

Set one carry-forward item:

- one top action for tomorrow morning
- one stop condition (what not to overdo)

This should directly feed Good Morning and Daily Ops planning.

### 4) Recovery cue (15-30s, optional)

Optional short reset cue:

- "What to let go of tonight" (one line)
- "When to resume tomorrow" (time window)

---

## Suggested output block

```md
### Good night closeout
- Day status: <finished well/partial/blocked>
- One signal: <single bullet>
- Tomorrow top action: <single action>
- Stop condition: <single boundary>
- Optional reset cue: <one line>
```

---

## Persistence guidance

- Write to a non-identity operational surface, for example:
  - `users/<id>/self-memory.md` (ephemeral)
  - or a dedicated daily note path.
- If a signal is identity-relevant, stage it as a **suggestion** for gate review; do not merge directly.

---

## Modes

| Mode | Duration | Use case |
|------|----------|----------|
| Minimal | ~1-2 min | Low-energy nights |
| Standard | ~2-4 min | Most nights |
| Reflective | ~4-6 min | End of sprint/week |
